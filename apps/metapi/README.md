# Metapi

Metapi 是一个面向多 AI 中转站聚合场景的统一网关，适合在 1Panel 中部署为集中入口，用于统一转发、令牌管理与定时任务调度。

## 应用简介

该应用将多个 AI 中转服务的访问能力收敛到一个统一网关内，方便在单一入口下完成鉴权、代理调用与日常维护。当前 1Panel 适配版本提供开箱即用的本地应用结构，部署后可直接通过 Web 端口对外提供服务。

## 功能亮点

- 聚合多个 AI 中转站，统一对外暴露访问入口
- 支持管理员令牌与代理令牌配置，方便分权管理
- 支持签到任务与余额刷新任务的 Cron 调度
- 支持数据目录持久化，便于保留运行数据与后续迁移
- 配置项集中在安装表单中维护，适合面板化运维

## 安装说明

安装时建议至少确认以下参数：

- 访问端口
- 数据目录
- `AUTH_TOKEN` 管理员令牌
- `PROXY_TOKEN` 代理令牌
- `CHECKIN_CRON` 自动签到计划
- `BALANCE_REFRESH_CRON` 余额刷新计划
- 时区配置

其中两类令牌都属于关键安全参数，建议直接使用高强度随机值。

## 默认配置

- 访问端口：`4000`
- 容器内部端口：`4000`
- 默认数据目录：`./data`
- 容器内数据目录：`/app/data`
- 默认签到 Cron：`0 8 * * *`
- 默认余额刷新 Cron：`0 * * * *`
- 默认时区：`Asia/Shanghai`

## 使用建议

- 首次安装时优先更换为高强度 `AUTH_TOKEN` 与 `PROXY_TOKEN`
- 若服务器已有统一反向代理，可将 Metapi 作为内部网关服务接入
- Cron 任务建议按实际中转站频率与额度策略调整，避免过于激进
- 数据目录建议放到稳定存储位置，便于备份与故障恢复

## 安装后检查

1. 打开 `http://<服务器IP>:<访问端口>` 确认服务已启动。
2. 检查令牌配置是否生效。
3. 检查数据目录是否已正确挂载。
4. 观察签到与余额刷新任务是否按预期执行。
